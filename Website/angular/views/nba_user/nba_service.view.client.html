NBA Rest API

Why NBA API you ask ??

This blog/API post is primarily meant for sports enthusiasts like myself to have easy access to NBA data. During my obsession with sports analytics, I tired in vain to find good API sources to collect all the data. However all the good API's were either discontinued; like ESPN or charging over 200$ for previous season data.

I could not afford any of these so as a good programmer, wrote a lot of python scripts to collect, clean and insert the data to SQL DB. So far, I have collected data from the 2014 season till 2016 season and the API will contain this data.

As I am learning about restful API design, this project was primarily meant as a way to learn about API design and maybe help some sports geeks out in the process.

API Features

The API currently can take two parameters i.e., "Date" or "player name". The response only supports XML format and I will be adding Json format soon.

https://safe-sands-52168.herokuapp.com/NBA-Logs?name=\NAME\

Lists the game logs for a player starting from the games in 2014. All the player names can be accessed, by passing '?name=' in the URL. The first and last names are separated by '-' in the URL. For example to get the game logs of Kyle Lowry, the link would be
https://safe-sands-52168.herokuapp.com/NBA-Logs?name=Kyle-Lowry

I higly recommend first getting all the names of the players and then iteratively collecting the game logs of every player.

https://safe-sands-52168.herokuapp.com/NBA-Logs/date?date=\DATE\

Lists the player logs for that particular date. Date format is MM/DD/YYYY. This is perfect when you want to get the dates for specific days of the year or specific months of the year.
